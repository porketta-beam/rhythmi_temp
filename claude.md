## 프로젝트 개요
- 목적: 키오스크에서 사용자가 간단한 설문을 통해 피부 타입을 파악하고, 그에 맞는 추천 루틴 및 제품을 제안하는 인터랙티브 경험을 제공한다.
- 입력 방식: 터치 기반 키오스크 UI.
- 전체 소요 시간: 10분 이내.
- 결과 데이터는 현재 화면에서만 노출하며 저장/전송하지 않으나, 추후 저장 및 분석 기능 확장 가능성을 고려한다.

## 기술 스펙 & 실행 환경
- 프런트엔드: Next.js + Tailwind CSS.
- 백엔드: 프로토타입 단계에서는 미구현, 향후 확장 예정.
- 통신: 기본적으로 온라인 연결 가능한 키오스크를 가정.
- 레이아웃: 기본 해상도 1080x1920 세로형 기준, 상·하 여백을 넉넉히 두고 반응형 동작(비율 변동 시 주요 요소가 중앙 정렬 및 축소/확대).
- 코드 구조는 이후 백엔드 연동 시 API 모듈 및 상태 관리 로직을 쉽게 추가할 수 있도록 Next.js의 라우팅 구조와 컴포넌트/훅 단위로 분리해 설계한다.

## 사용자 플로우
1. 시작 화면(`페이지1`)
   - 요소: 브랜드 로고, 간단한 서비스 소개 문구, `시작하기` 버튼.
   - 동작: 사용자가 `시작하기` 버튼을 터치하면 다음 페이지로 이동.
2. 정보 활용 동의(`페이지2`)
   - 요소: 개인정보·피부 데이터 활용에 대한 안내 문구, `동의`와 `거부` 버튼.
   - 동작:
     - `동의`: 질문 페이지로 이동.
     - `거부`: 플로우 중단 안내 후 시작 화면으로 돌아가기.
3. 질문 페이지(`페이지3-x`, 10~12문항 권장)
   - 형식: 한 화면에 한 문항 출력, 선택지는 2개 이상 단일 선택 버튼 스타일.
   - 콘텐츠: `client_requirements_251107.md`의 문항을 참고하되, 사용자가 10분 이내에 완료할 수 있도록 핵심 질문만 선별하여 구성. 질문 수는 유연하게 조정 가능.
   - 부가 요구:
     - 질문 진행도 표시 (예: "3 / 10")(선택사항)
     - 이전/다음 버튼 제공. 첫 페이지에서는 `이전` 비활성화.
     - 응답 선택 이전에는 `다음` 버튼 비활성화.
4. AI 생각 중 화면(`페이지4`)
   - 노출 타이밍: 각 질문 완료 후 결과 계산 직전에 1초간 노출.
   - 요소: 애니메이션 또는 로딩 인디케이터, "AI가 결과를 분석 중입니다"와 유사한 문구.
   - 동작: 1초 타이머 종료 후 자동으로 다음 페이지로 이동.
5. 결과 페이지(`페이지5`)
   - 요소:
     - 사용자의 피부 타입 라벨.
     - 간단한 설명 1~2문장.
     - 핵심 케어 포인트 2~3개.
     - 추천 루틴 간략 안내.
   - 데이터 매핑: `client_requirements_251107.md`의 "결과 유형" 섹션은 참고용으로만 활용하며, 실제 결과는 사용자 경험에 맞게 간결하게 구성.
   - CTA: `다시 시작하기` 버튼 → 시작 화면으로 이동.

## 기능 요구사항
- 질문 응답 데이터는 프런트엔드 단에서만 임시 유지. 외부 전송 없음.
- 결과 계산 로직(예시):
  1. 각 질문에 사전 정의된 가중치 및 타입 매핑을 구성.
  2. 응답 데이터 기반으로 가장 높은 점수를 받은 결과 유형을 선택.
  3. 동점일 경우 사전에 정의된 우선순위에 따라 결정.
- 결과 매핑 로직은 기획/디자인/개발이 공동으로 설계하며, 타입별 가중치 테이블과 우선순위 규칙을 별도 문서화한다.
- 페이지 전환은 터치 인터랙션에 즉각 반응해야 하며, 로딩 화면 외에는 인위적 지연 없음.
- 페이지 전환은 터치 인터랙션에 즉각 반응해야 하며, 로딩 화면 외에는 인위적 지연 없음.
- 중간에 사용자가 플로우를 종료하고 싶을 때를 대비하여 모든 페이지에 `시작 화면으로 돌아가기` 옵션을 제공 (결과 페이지 제외).
- 향후 데이터 저장 및 AI API 연동을 고려해, 상태 관리와 API 호출 레이어를 추상화한 구조를 선호한다.

## UI/UX 가이드
- 화면 비율: 키오스크 해상도 1080x1920(세로형) 기준으로 디자인. 1080x1920 이외의 해상도에서는 주요 요소가 중앙 정렬 및 축소/확대되도록 디자인.
- 터치 영역: 버튼 및 선택지는 최소 48px 이상의 터치 영역 확보.
- 타이포그래피: 큰 본문 텍스트(18~20pt), 선택지 18pt 이상.
- 색상: 각 결과 유형별로 상징색 사용 가능하나 브랜드 가이드라인 우선.
- 접근성: 대비비율 4.5:1 이상 유지, 로딩 화면에서 모션은 1초 이내의 부드러운 애니메이션 사용.

## 시스템 메시지 예시
- 시작 화면: "10분 안에 나만의 피부 루틴을 찾아보세요."
- 정보 활용 동의: “서비스 제공을 위해 피부 관련 정보 사용에 동의해 주세요.”
- 질문 진행 안내: "현재 5 / 10 질문 중입니다."
- 로딩 화면: “AI가 답변을 분석하는 중입니다…”
- 결과 페이지: “당신의 피부 타입은 ‘건조한 사무실형’입니다.”

## 디자인 & 자산 관리
- 현재 브랜드 컬러, 폰트, 로고 가이드가 없으므로 추후 전달 시 빠르게 교체할 수 있도록 Tailwind 테마 토큰과 CSS 변수 중심으로 스타일 구성.
- 공통 UI 컴포넌트(버튼, 카드, 진행바 등)는 별도 디렉터리(`components/common`)에 모듈화.
- 이미지/아이콘 자산은 `public/assets` 하위에 카테고리별로 정리하고, 변경 용이성을 위해 파일명/사용 위치 매핑표를 유지.

## 오류/예외 처리
- 네트워크 미사용을 가정하므로 서버 통신 오류 없음.
- 필수 입력 누락:
  - 질문페이지에서 응답 없이 `다음` 버튼 터치 시 “응답을 선택해 주세요” 토스트 또는 강조 표시.
- 타이머 실패:
  - 로딩 페이지 1초 타이머가 완료되지 않을 경우 최대 3초 후 강제 결과 페이지 이동.
- 결과 매핑 실패:
  - 매칭되는 결과 유형이 없을 경우 기본 유형(예: “예민 장벽 약한형”)으로 폴백하고 로그 남김.
