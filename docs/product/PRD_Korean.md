# 제품 요구사항 문서: eventManager

**버전:** 1.0
**최종 업데이트:** 2025-10-01
**문서 담당자:** 제품팀

---

## 요약

### 제품 개요
eventManager는 소규모 커뮤니티를 위해 특별히 설계된 모바일 우선 이벤트 관리 플랫폼입니다. 복잡하고 비싼 엔터프라이즈급 솔루션과 달리, eventManager는 누구나 스마트폰으로 직접 사용할 수 있는 필수 이벤트 관리 도구를 제공합니다.

### 해결하고자 하는 문제
소규모 사업체, 동아리, 친목 모임, 교육 기관은 참여자 추적, 현장 출석 체크, 경품 추첨, 회비 수집 등의 이벤트 관리 기능이 필요하지만, 기존 솔루션은 두 가지 극단으로 나뉩니다:

- **기본 도구** (엑셀, 구글 폼): 수동 작업, 오류 발생 가능성 높음, 실시간 기능 부족
- **엔터프라이즈 플랫폼** (Eventbrite, Cvent): 지나치게 복잡하고 비싸며, 대규모 컨퍼런스용으로 설계됨

이로 인해 **10-500명 규모의 커뮤니티에 적합한 솔루션이 부재**한 상황입니다.

### 솔루션
사용자 니즈에 따라 성장하는 2단계 플랫폼:

**1단계 (무료 티어):** 사용자 확보를 위한 핵심 참여자 관리 기능
**2단계 (프리미엄 티어):** 특정 커뮤니티 유형에 맞춤화된 부가 기능

### 성공 비전 (6개월)
입소문을 통한 확산으로 eventManager가 소규모 커뮤니티 이벤트의 기본 선택지가 됩니다. 사용자들은 필요에 따라 자연스럽게 프리미엄 기능으로 전환하며, 지속 가능한 수익 흐름을 창출합니다.

## 시장 분석

### 시장 기회

**목표 시장 규모:**
- 2025년 기준 58%의 팀이 소규모 이벤트를 더 많이 개최
- 전 세계 수백만 개의 소규모 커뮤니티가 적절한 도구 부족
- 대규모 컨퍼런스에서 소규모 반복 커뮤니티 모임으로 트렌드 전환

**현재 시장 공백:**
1. **모바일 네이티브 단순성:** 기존 도구들은 데스크톱 워크플로우 필요
2. **합리적인 가격:** 엔터프라이즈 도구는 소규모 팀에 월 $50-200 청구
3. **커뮤니티 중심:** 반복적이고 관계 중심적인 이벤트에 최적화된 솔루션 부재
4. **점진적 가격 책정:** 전부 아니면 전무 식의 가격 모델이 소규모 예산에 맞지 않음

### 경쟁 환경

**직접 경쟁사:**

| 경쟁사 | 타겟 | 가격 | 약점 |
|--------|------|------|------|
| RSVPify | 소중형 이벤트 | $27-63/월 | 데스크톱 중심, 복잡한 설정 |
| Partiful | 소셜 모임 | 무료 + 수수료 | 제한된 기능, 미국 전용 |
| Eventbrite | 모든 규모 이벤트 | 무료 + 3.7% 수수료 | 거래 수수료 누적 |
| WildApricot | 회원 조직 | $60-240/월 | 소규모 그룹에 비쌈 |

**간접 경쟁사:**
- **구글 폼 + 스프레드시트:** 소규모 주최자의 67%가 여전히 이 수동 방식 사용
- **카카오톡/왓츠앱 그룹:** 커뮤니케이션 중심, 관리 도구 부족
- **종이 기반 시스템:** 현장 출석 체크에 여전히 널리 사용

<image>
eventManager를 "간단함 + 저렴함" 사분면에 배치하고, 경쟁사들은 "복잡함 + 비쌈"(엔터프라이즈 도구) 또는 "간단함 + 제한적"(폼/스프레드시트) 사분면에 배치한 경쟁 포지셔닝 맵
</image>

### 차별화 요소

**3가지 핵심 기둥:**

1. **모바일 우선 설계**
   - 노트북 없이 모든 기능 완결
   - 한 손으로 현장 관리 최적화

2. **점진적 가격 책정**
   - 단순 체험판이 아닌 진정한 가치를 제공하는 무료 티어
   - 실제로 사용하는 기능에 대해서만 지불

3. **커뮤니티 맞춤화**
   - 수직 특화 기능(스포츠 리그, 스터디 그룹, 네트워킹 이벤트)
   - 만능 범용 도구가 아닌 맞춤형 솔루션

## 목표 사용자

### 주요 사용자: 이벤트 주최자

**인구통계:**
- 연령: 25-45세
- 역할: 동아리 회장, 소규모 사업주, 커뮤니티 리더, 교육 코디네이터
- 기술 숙련도: 모바일 앱 사용 가능, 기술 전문가 아님
- 이벤트 빈도: 연간 4-12회
- 이벤트 규모: 10-500명 참여자

**사용자 페르소나:**

**페르소나 1: 사라 - 북클럽 주최자**
- 월간 20-30명 모임 관리
- 현재 문제점: 카카오톡으로 참석 확인, 수동으로 명찰 제작
- 필요 사항: 빠른 출석 체크, 간단한 참석자 명단, 가끔 경품 추첨
- 예산: $0-15/월

**페르소나 2: 데이비드 - 소규모 사업주**
- 분기별 고객 감사 행사 개최(100-200명)
- 현재 문제점: Eventbrite 사용 중이지만 3.7% 수수료가 부담
- 필요 사항: 전문적 외관, 결제 수집, 브랜드 페이지
- 예산: $30-50/월

**페르소나 3: 김 교수 - 학술 워크숍 코디네이터**
- 세미나 및 심포지엄 주최(50-150명)
- 현재 문제점: 엑셀 스프레드시트, 인쇄된 명찰, 수동 출석 체크
- 필요 사항: 명찰 출력, 세션 추적, 익명 참여자 옵션
- 예산: 정당한 도구를 위한 부서 예산 사용 가능

### 부차 사용자: 이벤트 참여자

**사용자 경험 요구사항:**
- 등록을 위해 앱 설치 불필요
- 간단한 등록 절차(2분 이내)
- 모바일 친화적인 확인 및 업데이트
- 데이터 사용에 대한 명확한 안내

## 제품 기능

### 1단계: 무료 티어 (사용자 확보)

**목표:** 실제 문제를 해결하는 진정으로 유용한 무료 기능으로 시장 입지 확보

#### 1.1 이벤트 생성 및 관리

**기능:**
- 기본 정보로 무제한 이벤트 생성(이름, 날짜, 위치, 설명)
- 참여자 정원 설정
- 이벤트 상태 관리(초안, 게시됨, 진행 중, 완료)
- 반복 활동을 위한 과거 이벤트 복제

**사용자 플로우:**
```
앱 열기 → 이벤트 생성 → 세부사항 입력 → 공유 가능 링크 생성 → 완료
```

<image>
이벤트 이름, 날짜/시간, 위치, 최대 참여자, 설명 필드가 있는 간단한 이벤트 생성 폼을 보여주는 모바일 화면 목업
</image>

#### 1.2 참여자 등록 및 관리

**기능:**
- 공유 가능한 등록 링크(참여자 로그인 불필요)
- 맞춤형 등록 폼 필드(주최자가 수집할 정보 정의)
- 검색 및 필터링 가능한 실시간 참여자 목록
- 참여자 데이터 내보내기(CSV, Excel)
- 익명 참여자 옵션(닉네임 기반 등록)

**주요 특징:**
- **유연한 데이터 수집:** 주최자가 필수 필드 선택(이름, 연락처, 소속, 맞춤 질문)
- **개인정보 제어:** 식별 모드와 익명 모드 간 전환
- **즉시 업데이트:** 참여자 목록이 실시간으로 여러 기기에 동기화

<image>
분할 화면: 왼쪽 - 참여자 관점의 등록 폼; 오른쪽 - 주최자의 검색 바 및 필터 옵션이 있는 참여자 목록
</image>

#### 1.3 현장 출석 체크

**기능:**
- 이름 기반 빠른 출석 체크(검색 후 탭)
- 출석 통계(총 등록, 출석 완료, 불참)
- 오프라인 모드(연결 복구 시 동기화)
- 여러 기기에서 동시 출석 체크 가능

**사용자 플로우:**
```
이벤트 당일 → 이벤트 열기 → 출석 체크 모드 → 이름 검색 → 탭하여 확인 → 상태 업데이트
```

<image>
큰 검색 바, 출석 체크 버튼이 있는 참여자 이름, 상단에 "42/80 출석 완료"를 표시하는 진행 바가 있는 출석 체크 인터페이스 모바일 화면
</image>

#### 1.4 경품 추첨 시스템

**기능:**
- 등록자 또는 출석자 중에서 당첨자 추첨
- 몰입감 있는 공개를 위한 애니메이션 랜덤 선택
- 중복 당첨 방지(선택적 토글)
- 추첨 이력 로그
- 발표 모드(전체 화면 당첨자 표시)

**사용 사례:**
- 네트워킹 행사의 문전 경품
- 모금 행사의 래플 추첨
- 자원봉사 업무 무작위 선정
- 경품 대회

<image>
회전 애니메이션이 있는 경품 추첨 화면, 이어서 색종이 효과와 큰 이름 표시가 있는 당첨자 발표 화면
</image>

**1단계 제한사항 (무료 티어):**
- 동시 활성 이벤트 최대 3개
- 이벤트 종료 후 90일간 참여자 데이터 저장
- 기본 이메일 알림만 제공
- 표준 브랜딩에 "Powered by eventManager" 푸터 포함

### 2단계: 프리미엄 티어 (수익화)

**목표:** 특정 커뮤니티 니즈를 해결하고 지불을 정당화하는 맞춤형 프리미엄 기능 제공

#### 2.1 현장 명찰 출력 서비스

**기능:**
- 사전 디자인된 명찰 템플릿(10개 이상 스타일)
- 로고 및 브랜딩을 포함한 맞춤 명찰 디자인
- 향상된 출석 체크를 위한 QR 코드 통합
- **하드웨어 대여 옵션:** 이벤트 장소로 휴대용 열전사 프린터 배송

**가격 모델:**
- **소프트웨어만:** $15/이벤트 (자체 프린터 사용)
- **하드웨어 대여:** $50/이벤트 (프린터 배송 및 회수 포함)
- **명찰당 출력:** 대량 이벤트의 경우 $0.50/명찰

**기술 통합:**
- 휴대용 열전사 프린터와 호환(Brother QL 시리즈, Dymo LabelWriter)
- 여러 출석 체크 스테이션을 위한 인쇄 대기열 관리
- 출석 체크 시 즉시 명찰 출력

<image>
로고 업로드, 필드 선택(이름, 소속, 역할), QR 코드 토글, 색상 구성표 등의 맞춤화 옵션을 보여주는 명찰 디자인 인터페이스. 오른쪽에 실제 명찰 미리보기.
</image>

#### 2.2 맞춤형 브랜드 이벤트 페이지

**기능:**
- "Powered by eventManager" 브랜딩 제거
- 맞춤 도메인 지원(events.yourcompany.com)
- 로고가 포함된 브랜드 이메일 커뮤니케이션
- 맞춤형 색상 구성 및 스타일링
- 기존 웹사이트에 등록 페이지 임베드

**가격:** $25/월 또는 $15/이벤트

**대상 사용자:** 기업, 전문 조직, 반복 이벤트 주최자

<image>
일반 이벤트 페이지와 회사 색상, 로고, 도메인이 적용된 완전 브랜드 맞춤 페이지의 전후 비교
</image>

#### 2.3 수직 특화 애드온

**커뮤니티 유형별 모듈형 기능:**

**스포츠/레크리에이션 커뮤니티용:**
- 토너먼트 대진표 생성
- 팀 관리 및 일정 관리
- 점수 추적 및 리더보드
- **가격:** $20/월

**전문가/네트워킹 이벤트용:**
- 연사/세션 관리
- 시간대별 일정 생성 도구
- 참석자 네트워킹 기능(선택적 연락처 공유)
- **가격:** $30/월

**모금/유료 이벤트용:**
- 토스 페이먼츠를 통한 통합 결제 수집
- 자동 회비 알림
- 결제 상태 추적
- 영수증 생성
- **가격:** $20/월 + 2.5% 거래 수수료

**교육/워크숍 이벤트용:**
- 다중 세션 출석 추적
- 수료증 생성
- 피드백/평가 폼
- 자료 배포
- **가격:** $25/월

<image>
다양한 애드온 패키지(스포츠, 전문가, 모금, 교육)의 포함 기능에 체크마크가 표시된 기능 매트릭스 표
</image>

#### 2.4 고급 분석 및 보고

**기능:**
- 여러 이벤트에 걸친 출석 추세
- 참여자 유지율 분석
- 불참률 추적
- 맞춤형 보고서 생성
- 고급 필터링을 통한 데이터 내보내기

**가격:** 모든 프리미엄 구독에 포함

#### 2.5 프리미엄 지원 및 맞춤화

**고유한 니즈를 위한 엔터프라이즈 옵션:**
- 특정 커뮤니티 요구사항을 위한 맞춤형 기능 개발
- 전담 계정 관리자
- 우선 지원(4시간 응답 시간)
- 화이트 라벨 솔루션 옵션
- **가격:** 맞춤 견적(월 $200부터 시작)

## 사업 모델 및 수익화 전략

### 수익원

### 가격 철학

**무료 티어 전략:**
- 제한된 기능이 아닌 진정한 가치 제공
- 사용자 확보 및 입소문에 집중
- 제한 방식: 기능 제한이 아닌 이벤트 볼륨(활성 이벤트 3개)
- 목표: 무료 티어에서 70% 유지율 = 강한 제품-시장 적합성

**프리미엄 전환 트리거:**
- 자연스러운 니즈 진화(예: 첫 유료 이벤트 → 결제 수집 필요)
- 전문적 외관 요구사항(브랜딩)
- 특정 수직 니즈(스포츠 대진표, 연사 관리)
- 빈도 기반(연간 4개 이상 이벤트 = 구독이 합리적)

**예상 전환 퍼널:**
```
신규 사용자 100명 (무료 티어)

→ 프리미엄 전환 10명 (니즈 기반)
→ 상위 티어 업그레이드 2명 (특화/엔터프라이즈)
```

## 기술 아키텍처

### 기술 스택 (팀과 결정 예정)

**핵심 기술 요구사항:**

**1. 모바일 우선 개발**
- 네이티브 앱(iOS/Android) 또는 프로그레시브 웹 앱(PWA)
- 신뢰할 수 없는 인터넷 환경에서 현장 사용을 위한 오프라인 기능
- 스마트폰 한 손 사용에 최적화된 반응형 디자인

**2. 실시간 동기화**
- 여러 기기가 동일한 이벤트를 동시에 관리 가능
- 출석 체크 업데이트가 모든 주최자 기기에 즉시 표시
- 동시 출석 체크를 위한 충돌 해결

**3. 확장성 고려사항**
- 단일 이벤트에서 500명 동시 출석 체크 지원
- 6개월 마일스톤에서 월 10,000개 이상 이벤트 처리
- 빠른 참여자 검색을 위한 데이터베이스 설계(1초 이내)

**4. 외부 통합**
- **결제 처리:** 토스 페이먼츠 API (초기 한국 중심)
- **알림:** 확인 및 리마인더를 위한 이메일/SMS
- **캘린더:** 구글 캘린더, 애플 캘린더, 아웃룩으로 내보내기
- **프린터 하드웨어:** 블루투스/WiFi 열전사 프린터 연결

- 익명 모드: 참여자가 닉네임 사용, 개인정보 수집 안 함
- GDPR/개인정보 보호 규정 준수를 위한 내보내기 및 삭제 기능

#### 데이터 저장 및 보관

**무료 티어:**
- 이벤트 종료 후 90일간 참여자 데이터 보관

- 보관 기간 후 자동 삭제

**프리미엄 티어:**
- 연장 보관(1년 또는 활성 구독 시 무기한)

#### 개인정보 보호 및 규정 준수

**참여자 권리:**
- 참여자가 데이터 삭제 요청 가능
- 등록 중 명확한 동의
- 데이터 수집 전 개인정보 처리방침 공개

**보안 조치:**
- 암호화된 데이터 전송(TLS 1.3)
- 저장 시 데이터 암호화
- 정기적인 보안 감사
- GDPR/개인정보 보호법 준수

**익명 모드 기능:**

## 사용자 경험 및 디자인 원칙

### 디자인 철학

**1. 한 손 조작**
- 엄지 손가락으로 모든 핵심 기능 접근 가능
- 큰 탭 타겟(최소 44×44 픽셀)
- 하단 정렬된 주요 액션

**2. 3-탭 규칙**
- 홈 화면에서 3번 탭 이내에 모든 핵심 기능 수행 가능
- 예: 이벤트 생성 → 이름 입력 → 링크 공유 = 3탭

**3. 오프라인 우선**
- 인터넷 연결 없이 핵심 기능 작동
- 동기화 상태의 명확한 시각적 표시
- 오프라인 시 우아한 성능 저하

**4. 즉각적인 피드백**
- 출석 체크 확인 애니메이션
- 실시간 참여자 수 업데이트
- 로딩 상태가 2초 인지 시간을 초과하지 않음

### 주요 사용자 플로우

#### 플로우 1: 첫 이벤트 생성

```
1. 앱 다운로드 / 웹사이트 열기
2. 가입(이메일 + 비밀번호 또는 소셜 로그인)
3. 튜토리얼 오버레이(3개 화면, 건너뛰기 가능)
4. "첫 이벤트 만들기" 버튼
5. 이벤트 생성 폼(6개 필수 필드)
6. 미리보기 및 확인
7. 공유 링크 생성 → 복사 또는 직접 전송
8. 다음 단계가 있는 확인 화면
```

**완료 시간:** 3분 미만

## 성공 지표 및 KPI

### 1단계 성공 기준 (1-6개월)

**사용자 확보 지표:**
- **주요:** 6개월까지 등록 사용자 5,000명
- **부차:** 4개월까지 주당 신규 사용자 200명
- **바이럴 계수:** 1.2 (각 사용자가 평균 1.2명 초대)

**참여 지표:**
- **이벤트 생성률:** 등록 사용자의 60%가 최소 1개 이벤트 생성
- **반복 사용:** 사용자의 40%가 6개월 내 2개 이상 이벤트 생성
- **참여자 볼륨:** 플랫폼을 통해 관리되는 총 참여자 50,000명

**제품-시장 적합성 지표:**
- **순추천지수(NPS):** 6개월까지 > 40
- **사용자 유지율:** 3개월 후 사용자의 50% 활성
- **입소문 성장:** 유료 마케팅 없이 신규 사용자의 30%가 추천으로 유입

### 2단계 성공 기준 (7-12개월)

**수익화 지표:**
- **무료-유료 전환:** 정기 사용자(3개 이상 이벤트)의 15%가 프리미엄 업그레이드
- **월간 반복 수익(MRR):** 12개월까지 $10,000
- **사용자당 평균 수익(ARPU):** $2.50 (무료 및 유료 혼합)

**기능 채택:**
- **명찰 출력:** 12개월까지 100개 이벤트에서 이 서비스 사용
- **결제 수집:** 처리된 거래 볼륨 $50,000
- **수직 애드온:** 프리미엄 사용자의 5%가 특화 애드온 선택

**사업 건전성:**
- **이탈률:** 유료 구독자 월 < 10%
- **고객 생애 가치(LTV):** > $100
- **고객 확보 비용(CAC):** < $20
- **LTV:CAC 비율:** > 3:1

### 측정 및 분석

**추적 구현:**
- 주최자를 위한 이벤트 분석 대시보드(조회수, 등록, 출석 체크)
- 제품 지표를 위한 내부 관리자 대시보드
- 기능 최적화를 위한 A/B 테스트 프레임워크
- 사용자 피드백 수집(이벤트 후 인앱 설문)

**출시 전술:**
- **베타 프로그램:** 초기 테스트 및 추천사를 위해 20개 커뮤니티 모집
- **커뮤니티 파트너십:** 핵심 커뮤니티에 첫 6개월 무료 프리미엄 기능 제공
- **콘텐츠 마케팅:** "[특정 커뮤니티 유형] 이벤트를 더 잘 조직하는 방법" 가이드

**성장 메커니즘:**
- **바이럴 루프:** 참여자가 "Powered by eventManager"를 보고 주최자가 됨
- **추천 프로그램:** 기존 주최자가 각 추천당 1개월 프리미엄 획득
- **온라인 커뮤니티:** 주최자 포럼에서 활발한 활동(Reddit, Facebook 그룹)
- **템플릿 라이브러리:** 일반 사용 사례를 위한 사전 구축 이벤트 템플릿(워크숍, 모임, 모금)

**채널 전략:**
- **유기적 소셜 미디어:** 성공적인 이벤트를 보여주는 사용자 생성 콘텐츠
- **파트너십:** 보완 도구와의 통합(커뮤니티 관리 플랫폼)
- **PR 홍보:** 소규모 비즈니스 및 커뮤니티 구축에 초점을 맞춘 출판물 타겟

### 3단계: 수익화 및 확장 (7-12개월)

**프리미엄 전환:**
- **인앱 넛지:** 사용자가 무료 티어 한계에 도달할 때 상황별 프롬프트
- **성공 트리거:** 첫 성공적인 이벤트 후 프리미엄 기능 제공
- **이메일 캠페인:** 기능 하이라이트 및 사용 사례 교육

**시장 확장:**
- **수직 특화:** 스포츠 리그, 전문 협회를 대상으로 한 마케팅
- **지리적 확장:** 초기 출시 이후 추가 시장을 위한 현지화
- **엔터프라이즈 홍보:** 맞춤/화이트 라벨 솔루션을 위한 직접 영업

- **피벗 준비 아키텍처:** 피드백에 따라 빠른 기능 조정이 가능한 모듈식 디자인
- **대체 채널:** 유기적 성장이 느리면 타겟 광고에 예산 할당(월 $500 테스트)
- **파트너십 가속화:** 커뮤니티 플랫폼과 배포 파트너십 확보

#### 리스크 2: 약한 무료-유료 전환

**시나리오:** 5% 미만의 전환율로 사업 모델 지속 불가능

**완화 전략:**
- **조기 수익화 테스트:** 7개월이 아닌 3개월에 프리미엄 기능 A/B 테스트
- **가격 실험:** 다양한 티어 구조 테스트($10, $15, $25 가격대)
- **가치 추가 번들링:** 여러 애드온을 고가치 패키지로 결합
- **프리미엄 조정:** 필요시 무료 티어 한계 강화(예: 이벤트 3개 대신 2개)

#### 리스크 3: 기술 확장성 문제

**시나리오:** 규모 또는 피크 이벤트 부하에서 플랫폼 불안정

**완화 전략:**
- **부하 테스트:** 공개 출시 전 1,000명 동시 출석 체크 시뮬레이션
- **인프라 여유:** 첫날부터 예상 용량의 10배로 설계
- **모니터링 및 알림:** 자동 알림이 있는 실시간 성능 모니터링
- **우아한 성능 저하:** 서버 연결 불가 시에도 핵심 출석 체크가 작동하는 오프라인 모드

### 중간 영향 리스크

#### 리스크 4: 경쟁 반응

**시나리오:** 기존 플레이어가 유사한 모바일 우선 제품 출시

**완화 전략:**
- **속도 우위:** 시장 입지 확립을 위해 4개월 내 MVP 출시
- **커뮤니티 해자:** 강력한 사용자 관계 및 브랜드 충성도 조기 구축
- **기능 혁신:** 고유한 수직 애드온으로 개발 속도 유지
- **전환 비용:** 데이터 이동성 및 통합으로 마이그레이션 장벽 구축

#### 리스크 5: 결제 통합 의존성

**시나리오:** 토스 페이먼츠 또는 다른 타사 서비스의 약관 또는 가격 변경

**완화 전략:**
- **다중 공급자 아키텍처:** 여러 프로세서를 지원하는 결제 모듈 설계
- **백업 통합:** 대안으로 Stripe 및 PayPal 통합 준비
- **명확한 조건:** 프리미엄 결제 기능을 의존성이 아닌 편의성으로 포지셔닝

- 접근성 준수 수준(WCAG 2.0 AA/AAA)

- 고객 지원 전략(인앱 채팅, 이메일, 도움말 센터)

- 데이터 보관 정책 및 법적 규정 준수
- 서비스 약관 및 개인정보 처리방침 초안

## 부록

### 용어집

**활성 이벤트:** 게시되어 등록을 받거나 현재 진행 중인 이벤트.

**익명 모드:** 참여자가 실명 대신 닉네임으로 등록하고 개인 데이터가 수집되지 않는 이벤트 구성.

**출석 체크:** 등록된 참여자를 이벤트 참석자로 표시하는 프로세스.

**무료 티어:** 핵심 기능 및 사용 제한이 있는 무료 구독 수준.

**주최자:** 이벤트를 생성하고 관리하는 주요 사용자 역할.

**참여자:** 이벤트에 등록하고 참석하는 개인(부차 사용자).

**프리미엄 티어:** 추가 기능 및 높은 사용 한도가 있는 유료 구독 수준.

**수직 애드온:** 특정 커뮤니티 유형(스포츠, 전문가, 모금, 교육)을 위해 설계된 특화 기능 모듈.
